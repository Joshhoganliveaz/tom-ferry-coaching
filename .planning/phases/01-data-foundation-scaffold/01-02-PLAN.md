Create `index.html` at the repo root as a single HTML file with embedded CSS and JS. Follow the Tenet Calculator pattern from the research doc. Include:

**HTML Structure:**
- DOCTYPE html, lang="en", charset UTF-8, viewport meta
- Title: "Live AZ Co | Project Tracker"
- Google Fonts preconnect + link tag for Source Serif 4 (400;600;700) and DM Sans (400;500;600;700) with display=swap
- `<header>` with h1 "Live AZ Co Project Tracker"
- `<main id="app">` with loading message
- `<footer>` with `<span id="timestamp">`

**CSS (embedded in style tag):**
- CSS custom properties on :root for brand tokens:
  --olive: #5C6B4F, --canyon: #8B6F5C, --gold: #C9953E, --cream: #FAF7F2, --charcoal: #2C2C2C
  --font-heading: 'Source Serif 4', Georgia, serif
  --font-body: 'DM Sans', system-ui, sans-serif
- Reset: *, *::before, *::after { margin:0; padding:0; box-sizing:border-box }
- body: font-family var(--font-body), background var(--cream), color var(--charcoal), min-height 100vh, line-height 1.6
- h1,h2,h3: font-family var(--font-heading)
- header: background var(--olive), color white, padding, text-align center
- footer: background var(--charcoal), color var(--cream), padding, text-align center, font-size small
- .loading: text-align center, padding, color var(--canyon)
- .error: text-align center, padding, color red
- .project-list: max-width 900px, margin auto, padding
- .project-item: background white, border-radius, padding, margin-bottom, box-shadow subtle
- .status-badge: inline-block, padding small, border-radius, font-size small, font-weight 600
  - .status-shipped: background var(--olive), color white
  - .status-in_progress: background var(--gold), color white
  - .status-parking_lot: background var(--canyon), color white

**JavaScript (embedded in script tag):**

1. `escapeHtml(str)` -- sanitize dynamic text using createElement/textContent pattern per CLAUDE.md conventions

2. `validateProject(project, index)` -- validate each project entry:
   - Check required string fields: name, description, status
   - Validate status against VALID_STATUSES
   - Validate adoption against VALID_ADOPTION (if present)
   - Validate autonomy against VALID_AUTONOMY (if present)
   - Validate revenueProximity against VALID_REVENUE (if present)
   - Check versions is array if present
   - Warn if shipped project has priority scores
   - Log warnings to console with format: `[Validation] Project N (name): message`

3. `renderPage(data)` -- render a simple project list for Phase 1:
   - Clear #app content
   - Create a container div with class project-list
   - For each project: render a div.project-item with name (h3), status badge (span with status-badge + status-{status} classes), version, and description
   - Use escapeHtml for all dynamic text
   - Store data.projects on window.__projects for Phase 2 to use

4. `renderFooterTimestamp(lastUpdated)` -- set #timestamp textContent to:
   "Data from: {lastUpdated} | Loaded: {new Date().toLocaleString()}"

5. `async function init()` -- the main pipeline:
   - fetch(`./projects.json?v=${Date.now()}`) with cache-busting
   - Check res.ok, throw on error
   - Parse JSON
   - Validate each project with validateProject()
   - Call renderPage(data)
   - Call renderFooterTimestamp(data.lastUpdated)
   - Catch block: render error message in #app

6. `document.addEventListener('DOMContentLoaded', init)`

Important: Use optional chaining (?.) and nullish coalescing (??) for defensive rendering. Show "unreleased" for missing currentVersion, handle null deployedUrl gracefully.
