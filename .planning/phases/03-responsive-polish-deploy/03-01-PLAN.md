<task type="auto">
  <name>Task 3: Deploy to GitHub Pages and verify data update workflow</name>
  <files>.nojekyll</files>
  <action>
After user confirms repo name from the checkpoint above, execute the following steps in order:

1. **Create .nojekyll file**: Create an empty `.nojekyll` file in the project root to prevent Jekyll processing on GitHub Pages.

2. **Create GitHub repo**: Run `gh repo create Joshhoganliveaz/{confirmed-repo-name} --public --source=. --push --description "Live AZ Co Project Tracker for coaching calls"`. This creates the repo, sets it as remote, and pushes all existing code.

3. **Push .nojekyll**: `git add .nojekyll && git commit -m "deploy: add .nojekyll to skip Jekyll processing" && git push origin main`

4. **Enable GitHub Pages**: Run `gh api --method POST /repos/Joshhoganliveaz/{confirmed-repo-name}/pages -f "source[branch]=main" -f "source[path]=/"`. If this returns an error about Pages already being enabled, that is fine (it may auto-enable on public repos).

5. **Wait for initial deploy**: Wait 60-90 seconds for the first GitHub Pages deploy (first deploy takes longer than subsequent ones).

6. **Verify site is live**: Derive the repo name from the git remote with `REPO=$(basename $(git remote get-url origin) .git)`, then run `curl -s -o /dev/null -w "%{http_code}" https://joshhoganliveaz.github.io/$REPO/` and confirm it returns 200.

7. **Verify data update workflow (DEPLOY-03)**: Make a trivial update to projects.json (e.g., update the timestamp or add a comment field), then sync the inline data in index.html using:
```bash
node -e "
const fs = require('fs');
const data = JSON.parse(fs.readFileSync('projects.json', 'utf8'));
const html = fs.readFileSync('index.html', 'utf8');
const updated = html.replace(
  /const PROJECT_DATA = .*?;/,
  'const PROJECT_DATA = ' + JSON.stringify(data) + ';'
);
fs.writeFileSync('index.html', updated);
console.log('Inline data updated.');
"
```
Then commit both files and push. Wait 60 seconds and curl the site again to confirm it returns 200.

If `gh repo create` fails with an auth error, pause and ask the user to authenticate with `gh auth login`.
