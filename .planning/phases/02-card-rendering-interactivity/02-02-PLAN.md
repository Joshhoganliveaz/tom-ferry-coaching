- `.sort-controls` div with a label "Sort:" and a `<select id="sort-select">` with two options: "Most Recently Deployed" (value="recent") and "Autonomy Level" (value="autonomy")

After setting innerHTML of #app (at the end of renderPage), attach event listeners:
- Add click listeners to all `.filter-tab` buttons that call `filterCards(button.dataset.filter)` and update active class
- Add change listener to `#sort-select` that calls `sortCards(select.value)`

**3. Add JS functions** (after the existing render functions, before renderPage):

`filterCards(status)`:
- Query all `.project-card` elements
- For each card: if status is 'all' or card.dataset.status matches status, remove 'hidden' class; else add 'hidden' class
- Update active tab: remove 'active' from all .filter-tab buttons, add 'active' to the one with matching data-filter
- Store current filter in a module-level variable `currentFilter` (initialize as 'all' at script top)

`sortCards(sortKey)`:
- Get the .project-grid container
- Get all .project-card elements as an array
- Sort the array:
  - If sortKey is 'recent': sort by data-deployed attribute (descending, most recent first). Cards with no deployed date (empty string or "null") sort to bottom.
  - If sortKey is 'autonomy': sort by data-autonomy attribute using an order map: fully_independent=3, needs_guidance=2, josh_dependent=1. Descending (highest first).
- Re-append sorted cards to container using appendChild (moves DOM nodes, does not clone)
- Store current sort in a module-level variable `currentSort` (initialize as 'recent' at script top)

Also add `data-deployed` and `data-autonomy` attributes to each .project-card in the renderCard() function:
- `data-deployed="${project.deployedDate || ''}"`
- `data-autonomy="${project.autonomy || ''}"`

IMPORTANT: Use escapeHtml() on any dynamic values inserted into HTML. The data-* attribute values come from validated enum/date fields so they are safe, but any visible text must be escaped.
